# 部署上线清单

这份清单面向“别人也能访问”的正式部署，而不是只在你电脑本地运行。

## 一、最省事的上线方案

适合先出一个可访问的网址给别人体验。

需要准备：
- GitHub 账号
- Vercel 账号
- 你的项目代码仓库
- 一个域名
- DeepSeek 聊天 API key
- 火山方舟文生图 API key

部署步骤：
1. 把 `tongqu-growth-web` 上传到 GitHub。
2. 在 Vercel 导入这个仓库。
3. 在 Vercel 项目环境变量里填写这些值：
   - `OPENAI_API_KEY`
   - `OPENAI_BASE_URL`
   - `OPENAI_MODEL`
   - `VOLCENGINE_ARK_API_KEY`
   - `VOLCENGINE_ARK_BASE_URL`
   - `VOLCENGINE_ARK_IMAGE_MODEL`
   - `NEXT_PUBLIC_SITE_URL`
4. 触发部署。
5. 绑定你自己的域名。

部署成功后可检查：
- `/`
- `/adventure`
- `/teachers`
- `/api/health`

## 二、当前版本上线后具备的能力

- 儿童互动故事页
- 快捷选项互动
- 浏览器语音输入
- 浏览器语音播报
- 章节插图生成按钮
- 三个轻量小游戏
- 老师和家长辅助内容生成页

## 三、当前版本还没有做的内容

这些不是不能上线，而是上线后仍然属于下一阶段增强项：

- 用户登录
- 数据库存档
- 勋章和成长记录持久化
- 图片云存储
- 音频云存储
- 后台管理系统
- 内容审核后台

## 四、如果你想长期运营而不是先演示

还建议补：
- 数据库
- 对象存储
- 接口限流
- 错误监控
- 访问统计
- 隐私政策页面
- 服务条款页面

## 五、如果主要给中国大陆用户访问

建议考虑：
- 国内云服务器或国内 Serverless
- 已备案域名
- 国内对象存储

注意：
- Vercel 上线快，但国内网络体验不一定一直稳定。
- 如果是比赛演示或先给少量人试用，Vercel 足够快。
- 如果后续要长期推广，更推荐国内部署。

## 六、你后面必须准备给我的东西

- 正式域名
- 正式项目名
- 首页主图或 logo
- 你想先用 Vercel 还是国内云
- 是否需要登录
- 是否要保存聊天记录和勋章

## 七、上线前的安全提醒

- 不要把 `.env.local` 上传到公开仓库。
- 正式上线前最好更换当前测试 key。
- 环境变量只能配在服务端，不要写到前端页面。
- 图片生成会消耗接口额度，正式上线前要确认配额和费用。
