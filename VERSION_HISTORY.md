## v1.1.3 - 闽食探索任务闭环与区域游戏优化版

- 时间：2026-04-26
- 定位：强化闽食成长岛的家园任务提交、幼儿表达、区域活动和家长延续
- 状态：已通过 `npm run lint`、`npx tsc --noEmit`、`npm run build`
- 是否已同步服务器：已同步，服务器提交 `5469ccc`

### 本次主要改动

- 将“闽食探索勋章墙”升级为“闽食家园任务提交区”，四张任务卡支持选择、输入、可选本机照片预览、提交反馈和成长打卡。
- “闽食小寻宝”“亲近美食章”“家庭小主厨”“闽食宣传员”提交后会写入成长记录，并用温和鼓励语反馈，不强迫进食。
- 儿童端当前闽食小游戏 3 改为“闽食小小播报员”，支持选择美食、提示卡组合、播放播报示范和完成打卡。
- 儿童端当前闽食小游戏 4 改为“泉州小厨房”，支持选择美食、按顺序点制作动作卡、播放小厨师口令和完成打卡。
- 旧 `peerEncourage`、`mealTray` 保留历史兼容，老师端和家长端显示为历史同伴鼓励记录、历史午餐小餐盘记录。
- 老师端和家长端同步识别 `foodReporter`、`foodKitchen`，可生成活动方案、同步家庭任务和展示新奖章记录。
- AI prompt 增加小小闽食播报员、泉州小厨房、区域活动步骤、厨房口令和家庭小任务约束。
- 同步前总检修复儿童端成长记录册统计口径，勋章、小游戏和最近记录按当前幼儿身份过滤，不再混入其他幼儿记录。
- 同步时加固部署脚本，默认使用本机 SSH 别名，并把服务器健康检查扩展到首页、儿童端、冒险页、家长端、教师端和 `/api/health`，避免核心页面 500 被漏报。

- 时间：2026-04-26

## v1.0.0 - 第一版基线

- 时间：2026-04-26 17:55 +08:00
- 定位：童趣成长乐园第一版基线
- 状态：已通过 `npm run lint`、`npx tsc --noEmit`、`npm run build`
- 说明：此前零散改动归并为本基线版本，后续每次功能或文案调整都从小版本继续记录。

### 当前已纳入的主要能力

- 首页仅保留身份入口：儿童互动、教师辅助、家长端。
- 儿童端先选择幼儿小名牌，再进入幼习宝或闽食成长岛。
- 幼习宝包含习惯养成、安全判断、听绘本故事和语音播放能力。
- 闽食成长岛包含闽南食育互动、饮食偏好观察、餐盘营养提示和语音提示。
- 教师辅助台可管理花名册、查看游戏打卡汇总、调整小游戏文案、生成活动方案，并设置教师账号登录。
- 家长端通过幼儿身份查看老师同步内容、成长记录和反馈入口。

## 后续版本规则

- 小改动：递增补丁号，例如 `v1.0.1`、`v1.0.2`。
- 成组功能：递增小版本，例如 `v1.1.0`。
- 大幅重构：递增主版本，例如 `v2.0.0`。
- 每次改动后都在本文件追加：版本号、日期、改动点、验证结果、是否已同步服务器。

## 备份规则

- 当前基线源码压缩包放在本机 `version-backups/` 目录。
- 压缩包不包含 `node_modules`、`.next`、`.git`、本地密钥和运行日志。
- 后续需要回退时，优先按本文件找到版本号，再使用对应压缩包或 Git 提交记录恢复。
